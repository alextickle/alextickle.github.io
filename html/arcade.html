Arcade

Overview

I've found writing simple computer games to be an extremely effective way to practice applying programming concepts. I decided to combine a few of the games I've written into 'Arcade', which is a simple Express web application that serves four arcade-style single page applications: Asteroids, Battleship, Black Jack, and Turtle Graphics. Each of these SPAs were originally independent side projects that I worked on for varying amounts of time at different stages in my studies. They all posed their own unique technical challenges.

The view contains logic for handling user input and displaying data received from the model. Its implementation has been hidden from the model and controller components so that it could readily be swapped out for a different view if needed.

The MediaItem class defines fields and methods that are used by its child classes (CD, DVD, and Book). At the moment there is no significant difference between the three child classes, but perhaps in a future version of the program they could be updated with functionality that is specific to each media type.

The components communicate with each other by sending and editing Command objects, which are basically just MediaItems wrapped in objects so that can include info on the type of user request. The view handles user input and creates and appropriate Command object to pass to the controller, which in turn passes it to the Model. The model uses the Command object to update itself and then alerts the view that the update is complete. Then the view responds with a more specific request for data, which the model then exports to the view for display to the user.

The model stores all data in an ArrayList of Properties objects. It reads and writes data to a file (inventory.txt). If there is no data file in the current directory at runtime then the application creates a new one. In my submission files I’ve included an inventory.txt file that is prepopulated with data so that the application’s functionality can be more readily demonstrated. The inventory file is saved after each update to the model.
